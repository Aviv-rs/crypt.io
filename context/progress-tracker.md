## Architecture Decisions

- **TanStack Query for client data** — paged fetches, request dedup, cache, loading/error states are worth a single dep. Justification logged here per `ai-workflow-rules.md`.
- **Server pagination by page number** — deterministic for the URL-state model and easier to defend than cursor in an interview. Cost: `OFFSET` scans, acceptable on SQLite at this dataset size.
- **CSV (`.csv`) over hand-rolled `.xlsx` / OOXML zip** — assignment says "spreadsheet file"; CSV qualifies, opens natively in Excel/Numbers/LibreOffice, takes ~30 lines to implement honestly. Hand-rolling OOXML zip + sheet1.xml costs days for no product value.
- **Server code colocated inside the feature folder** (`src/features/transactions/server/`) instead of a top-level `src/server/`. Single feature → vertical slice owns schema + handler + (later) CSV streaming. README has no opinion on layout, and Bun's bundler only ships what `frontend.tsx` imports, so server files stay out of the browser bundle. Trade-off: relies on the convention that `components/` never imports `server/` — enforced by review, not by tooling.
- **Controller-only server module per route** — service/controller split would be 1:1 ceremony for these thin handlers. `getTransactions` parses, queries Drizzle, and responds in one file. If a handler grows non-trivial business logic, split a service back out.
- **Zod for query-param validation** — single schema gives coercion, defaults, clamping (`pageSize` max 200), and structured error issues for free. Pays back further when sort allowlist (Unit 4) and filter shape (Unit 6) extend the same schema.
- **Feature-based folder layout** (`src/features/transactions/`) — borrowed from the Inventiva take-home. Everything transactions-related lives in one folder; easy to find, easy to delete.
- **Generic shadcn token names + semantic status tokens** — token names follow shadcn convention (`--background`, `--card`, `--primary`, ...) plus `--success`, `--warning`, `--info`, `--destructive` for status. Avoids design-platform-specific naming leaking into the codebase.
- **React Compiler enabled** — auto-memoization removes the need for manual `useMemo`/`useCallback`. Wired via a local Bun bundler plugin (`react-compiler.bun-plugin.ts`) that runs `@babel/core` with `babel-plugin-react-compiler` + TypeScript/React presets on app `.{tsx,jsx}` (skips `node_modules`).
- ~~**No router in v1**~~ — reversed in Unit 7. With page / pageSize / sort / dir / filters all on the URL, hand-rolling URL sync was paying for itself in maintenance, not features. **Adopted TanStack Router**: code-based config (no Vite plugin), `validateSearch` runs the same Zod schemas the server uses, `useSearch` + `useNavigate` replace the would-be custom hook, and a `NotFound` route comes free. Cost: one extra dep (~30KB gzipped) and a small router boot file.
- **Local self-hosted variable fonts** (Inter + JetBrains Mono, regular + italic, in `src/assets/fonts/`). Avoids CDN runtime, works offline, no FOUT after first load. Trade-off: ~600KB shipped vs. external request.
- **Dark token values in `:root`** (deviating from shadcn convention of `:root` = light + `.dark` = overrides). Reason: v1 is dark-only; shipping a dead light palette is harder to defend than the deviation. Future light theme adds `.light { ... }` overrides + a toggle without touching consumers.
- **Sidebar shell over plain top header** — uses shadcn `SidebarProvider` + `Sidebar` primitives. Desktop is always-open (no trigger rendered); mobile uses the built-in Sheet via a single `SidebarTrigger` inside `TopBar` hidden on `md+`. Justification: matches Aurora layout reference; gives room for future nav items without a re-layout.
